🤖 AI By

AI By is a production-grade, plugin-first coding agent. The core stays minimal; everything else is a plugin, loaded only when needed.

Install 
# macOS / Linux / WSL
curl -fsSL https://raw.githubusercontent.com/simpletoolsindia/ai-coder/main/install.sh | bash

# Windows PowerShell
iwr -useb https://raw.githubusercontent.com/simpletoolsindia/ai-coder/main/install.ps1 | iex

# npm
npm install -g ai-by

Features

🔌Plugin-first
Every feature is a manifest-driven plugin. Load only what you need.
🧭PLAN / EXECUTE modes
Press Tab in the REPL to switch. PLAN mode permits only read-only tools.
🧠Persistent memory
Long-term memory persists across sessions, with a search index.
🗜️Adaptive compression
Auto-compact at 95% utilization; manual /compact command.
🔎Tool RAG
Tools are loaded dynamically, not dumped into the prompt.
🛡️Reliability
Retries, argument validation, idempotency, loop guard, completion check.
🔁Multi-provider
OpenAI, Anthropic, Gemini, Ollama, LM Studio, any OpenAI-compatible.
🪟Cross-platform
macOS, Linux, Windows. Single-line installer for every OS.
🧩MCP
Model Context Protocol adapter for in-process and stdio servers.
🤝Sub-agents
Delegate work to focused sub-agents with isolated prompts.
📊Live status
Emoji + progress bar updates per tool call (read, write, bash, …).
📚Self-learning
Writes AGENT.md with observed patterns and personalized hints.

Architecture

Core is intentionally minimal: EventBus, DI Container, Settings, Tool Registry, Plugin Manager, Provider Manager, Planner. Everything else is a plugin or built-in tool. The system prompt is built per-request with a 2-line OS / supported commands meta block so the LLM always knows the host environment.

Reliability

Every tool call goes through a resilientInvoke wrapper that:

  • validates arguments against the tool schema before invocation,
  • retries transient failures (network, timeout),
  • de-duplicates identical calls via an idempotency cache,
  • blocks unsafe tools in PLAN mode (read-only allow-list).

A LoopGuard detects repeated tool calls, oscillation, and no-progress text. A CompletionCheck verifies the agent actually finished the work before returning, retrying if not.
